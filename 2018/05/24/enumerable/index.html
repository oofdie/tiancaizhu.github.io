对象的可枚举属性与不可枚举属性

May 24, 2018
对象的属性就可枚举性来区分则分为可枚举属性与不可枚举属性。

如何区分对象属性的可枚举性

对象的 propertyIsEnumerable 方法可以判断该属性在此对象中是否可被枚举

如何设置属性的可枚举性

可通过 Object 的 defineProperty 或者 defineProperties 方法定义或修改对象属性的可枚举性

defineProperty

defineProperties

属性的可枚举性是如何体现的

    一个属性的可枚举性直接影响到该属性是否可以被遍历到，因此当我们用 for...in, Object.keys(), Object.values(), JSON.stringify()等存在直接或间接遍历对象的方法处理该对象时，
不可枚举属性会被直接过滤掉。



    值得思考的是这里我们在 instance 的构造函数的 prototype 上定义了一个属性 b，当我们用上述方法处理 instance 时出现了各方法间不能相互印证的情况,
其他的方法都将这个属性忽略掉了，而 for...in 却遍历到了这个属性，这是什么原因呢？

    如果我们用 hasownproperty 这个方法检测属性 b 时我们会发现 属性 b 并不是 instance 的自带属性，而且相对与 instance 来说 b 的可枚举性是 false



    这就可以解释为什么其他的方法会遍历不到 b 这个属性，那为什么 for...in 可以遍历到呢？？

    原来 for...in 在遍历时会沿原型链向上查找，当查找到 b 这个属性并且确定 b 相对于构造函数 obj 的原型 enumerable 为 true 时，b 就不会被忽略。。。。
